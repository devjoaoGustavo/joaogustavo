Experiências
  • Criar e desenvolver software para manter serviços de internet com qualidade na Locaweb

    Esta foi uma experiência essencial na minha carreira como programador Web. Durante os 3 anos que trabalhei alí, aprendi muitos dos conceitos de desenvolvimento e qualidade de software que tenho hoje. Fui par de desenvolvedores altamente capazes, e formávamos um time comprometido com a qualidade do código que escrevia. Pude contribuir com aquele entusiasmo de quem estava apenas iniciando sua carreira, mas que tinha o comprometimento e a força de vontade inabaláveis, e isso me fez crescer muito.

    Escrevia fundamentalmente APIs REST e cronjobs, na maior parte do tempo em Ruby, mas também em Java e Scala. E para dar vida aos sistemas, usava alguns serviços como PostgreSQL, Redis e RabbitMQ para persistência de dados ou troca de mensagens, o que permitia realizar processamentos assíncronos, como atualização de planos e cobranças recorrentes de clientes. Por fim, fiz parte de um time ágil, que usava principalmente conceitos de SCRUM, mas que fazia algumas adaptações para sua realidade.

  • Ajudar clientes nos processos fiscais de sistema ERP na Riosoft

    Alí dei meus primeiros passos no mundo da tecnologia, e estava super empolgado com tudo o que encontrava. Aprendi como poderia colocar em prática os conceitos de lógica de programação que eram ensinados na faculdade. Ganhei uma boa noção de abstração, importantíssima na implementação de soluções de software que atendam problemas reais. Tive a change de visitar muitos clientes para ajudá-los nos processos fiscais do sistema que eu ajudava a manter, e isso me deu grande empatia para pensar nos usuários que utilizariam meus programas. Conheci pessoas muito boas, que tinham uma forma de pensar e de revolver problemas muito diferente de tudo que eu havia visto até então, fiquei impressionado.

    Afim de entender muitos dos problemas dos clientes, precisava fazer consultas SQL em um banco de dados Microsoft SQL Server, utilizando o SQL Server Management Studio. Além das consultas, ajudava a manter muitas store procedures que faziam integrações entre diferentes módulos do sistema ERP, um monolito escrito no Delphi. Algumas funcionalidades, como a geração de arquivos para escrituração e apuração de impostos, eram feitas em bibliotecas externas ao sistema, mas também eram escritas na linguagem de programação Object Pascal, usando o Delphi
